# ETF 投资增强设计：评分改进 + 操作建议层

日期：2026-05-16

## 背景

用户画像：兼职股民，10-50万资金，3-5只均衡持仓，1-6月中线波段。

三大痛点：
1. **信号准确率不够**：扫描器看多但买入后仍亏
2. **区分度不够**：Top ETF 分数挤在一起，难选择
3. **缺少操作建议**：只说"看多"，不说何时买、买多少

## 方案概述

方案 B：评分增强 + 操作建议层。两步走：
- 第一步改进评分算法（趋势阶段、板块排名、风险调整），提升准确率和区分度
- 第二步新增操作建议层（入场区间、止损、目标、仓位、时机），直接可执行

数据来源全部基于现有 K线/资金流/IOPV 数据，无需新增数据源。

## 第一步：评分增强

### 1.1 趋势阶段检测（Trend Stage）

在 Phase 1 quick_score 之后、Phase 2 之前，对每只 ETF 判断趋势阶段：

**初期 (early)：**
- 价格刚站上 MA20，或 MA5 上穿 MA20
- RSI 从 <40 回升到 40-55
- 量能配合放大
- 评分乘数：1.0（信号可信度高，值得入场）

**中期 (mid)：**
- MA5/20/60 多头排列
- RSI 55-70
- 价格在 MA20 附近
- 评分乘数：1.0（信号持续性强，可持仓/加仓）

**末期 (late)：**
- 价格远离 MA20 >5%（已有 price_ext_penalty，此处理解为趋势生命周期晚期）
- RSI >70 或 MACD 顶背离
- 缩量上涨
- 评分乘数：0.5-0.7（追高风险大，应降权）

**实现细节：**
- 输出字段：`trend_stage: "early" | "mid" | "late"` + `trend_stage_multiplier: float`
- 趋势阶段作为乘数影响 combined_score
- 乘数范围：early=1.0, mid=1.0, late=0.5~0.7（根据末期程度连续插值）

### 1.2 板块内相对排名

watchlist.yaml 已有 10 个分类（宽基指数、科技、消费、医药等）。

**实现细节：**
- Phase 1 结束后，按 category 分组，计算每只 ETF 在板块内的百分位排名
- 输出字段：`sector_rank: "3/12"`, `sector_percentile: 75`
- **星级门槛调整**：同板块只有前 30% 可给三星，其余最多两星
  - 效果：科技板块 12 只 ETF 中只有前 3-4 只可获三星
  - 解决"Top 10 都差不多"的问题——板块内排名拉开差距

### 1.3 风险调整分

从 K 线数据计算三个风险指标：

| 指标 | 计算 | 用途 |
|------|------|------|
| ATR(14) | 近14日真实波动幅度均值 | 止损距离 + 波动率度量 |
| 近20日波动率 | 日收益率标准差 | 风险惩罚系数 |
| 近20日最大回撤 | 最高到最低跌幅 | 极端风险过滤 |

**风险调整公式：**

```
risk_penalty = 1.0 / (1.0 + volatility * risk_aversion)
risk_adjusted_score = raw_score * risk_penalty
```

参数：
- `risk_aversion` 默认 5.0（可通过 watchlist.yaml 配置）
- 效果示例：
  - 波动率 5%（稳健 ETF）→ risk_penalty ≈ 0.96，几乎无惩罚
  - 波动率 15%（中等）→ risk_penalty ≈ 0.57，适度惩罚
  - 波动率 25%（高波动）→ risk_penalty ≈ 0.44，大幅惩罚

**极端风险过滤：**
- 近20日最大回撤 > 15% 的 ETF，额外乘 0.7 惩罚

### 1.4 组合排名公式调整

**现有公式：**
```
combined = 0.3 * quick_score + 0.7 * deep_normalized + p1_bonus
```

**改为：**
```
combined = (0.3 * quick_score + 0.7 * deep_normalized + p1_bonus)
           * trend_stage_multiplier
           * risk_penalty
```

板块排名不进公式，但影响星级：
- 同板块前 30%：可给三星
- 同板块 30%-60%：最多两星
- 同板块后 40%：最多一星

## 第二步：操作建议层

### 2.1 入场区间（Entry Zone）

基于 K 线数据计算：

| 价格位 | 计算方法 |
|--------|---------|
| 强支撑 | 近20日最低点 |
| MA支撑 | MA20 当前值 |
| 当前价 | 最新收盘价 |

**入场策略：**
- 趋势初期 + 价格在支撑位附近 → `"immediate"`：可立即入场，区间 [强支撑, MA支撑]
- 趋势中期 + 价格在 MA20 上方 → `"pullback"`：等回踩 MA20 再入场，区间 [MA支撑*0.98, MA支撑*1.02]
- 趋势末期 → `"avoid"`：不建议新入场

输出字段：
```json
"trading_plan": {
  "entry_zone": {"low": 1.23, "high": 1.25, "current": 1.24},
  "entry_strategy": "pullback",
  "entry_detail": "等回踩1.23-1.25区间入场"
}
```

### 2.2 止损位（Stop Loss）

三档止损，取最合理者：

| 方法 | 计算 | 适用 |
|------|------|------|
| 固定止损 | 入场价 - 1.5 × ATR(14) | 通用 |
| MA止损 | MA20 下方 2% | 趋势明确时 |
| 结构止损 | 近20日最低点 - 0.5% | 支撑明确时 |

取三档中最高者作为止损位（更保守）。

输出字段：
```json
"stop_loss": {
  "price": 1.19,
  "method": "atr",
  "risk_pct": 4.2
}
```

### 2.3 目标价（Target）

基于风险收益比计算：

| 目标 | 计算 |
|------|------|
| TP1 | 入场价 + 2 × 风险（即 R:R = 2:1） |
| TP2 | 入场价 + 3 × 靠险（即 R:R = 3:1） |

如果 TP1/TP2 超过近期阻力位（近20日最高点），则用阻力位替代。

输出字段：
```json
"targets": {
  "tp1": {"price": 1.32, "method": "2R", "pct": 8.4},
  "tp2": {"price": 1.38, "method": "3R", "pct": 12.6}
}
```

### 2.4 仓位建议（Position Size）

基于信心度和波动率：

```
base_position = portfolio_total * base_pct  // base_pct 默认 20%
adjusted = base_position * confidence_factor * volatility_factor

confidence_factor: 
  - 三星 = 1.2
  - 两星 = 1.0
  - 一星 = 0.8

volatility_factor:
  - 低波动 (<10%) = 1.1
  - 中波动 (10-20%) = 1.0
  - 高波动 (>20%) = 0.8
```

最终仓位约束：单只 10-30%，总持仓 3-5 只。

输出字段：
```json
"position": {
  "pct": 22,
  "range": [18, 26],
  "reason": "三星+低波动，建议偏重仓位"
}
```

### 2.5 时机提示（Timing Hint）

综合趋势阶段、RSI、乖离率生成一句话提示：

| 趋势阶段 | RSI | 提示 |
|---------|-----|------|
| early | <55 | 可立即入场，趋势刚启动 |
| early | >55 | 关注回踩MA20入场机会 |
| mid | <65 | 持仓可加仓，注意仓位 |
| mid | >65 | 持仓不动，不加仓 |
| late | >70 | 减仓或设置跟踪止损 |
| late + 缩量 | 任意 | 量价背离，考虑止盈 |

输出字段：
```json
"timing": {
  "action": "pullback",
  "hint": "等回踩1.23-1.25区间入场，趋势初期信号可信度高"
}
```

## 实现优先级

| 优先级 | 功能 | 预期收益 |
|--------|------|---------|
| P0 | 趋势阶段检测 | 直接解决"末期高分"问题，提升准确率 |
| P0 | 操作建议层（入场/止损/目标） | 直接解决"不知道怎么操作"问题 |
| P1 | 板块内相对排名 | 解决区分度问题 |
| P1 | 风险调整分 | 让稳健标的排名上浮 |
| P2 | 仓位建议 | 需要用户输入总资金才能精确，默认按比例 |
| P2 | 时机提示 | 依赖趋势阶段，锦上添花 |

---

## 测试策略

1. 现有 golden test 体系保证不破坏已有评分
2. 新增趋势阶段测试：early/mid/late 三种情况的乘数计算
3. 新增风险调整测试：不同波动率的惩罚系数验证
4. 新增交易计划测试：入场/止损/目标的计算逻辑
5. 新增板块排名测试：分组排名和星级门槛

## 限制与风险

- 纯技术面模型无法预测黑天鹅/政策突变
- 回测验证需后续单独建设（当前无历史回测框架）
- 仓位建议默认按比例，精确计算需用户输入总资金
- 趋势阶段判断依赖 MA 和 RSI 参数，极端行情（如连续涨停）可能误判
