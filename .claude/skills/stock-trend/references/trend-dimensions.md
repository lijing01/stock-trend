### 1.7 K线形态分析

**检查项**: 最近 5-10 根K线的形态组合

详细识别标准、特征计算公式和判定阈值见 **[kline-patterns.md](kline-patterns.md)**。

**核心特征值**（快速参考）：
- 实体大小 = abs(close - open)
- 上影线 = high - max(open, close)
- 下影线 = min(open, close) - low
- 方向 = close > open 为阳线，close < open 为阴线

**看多反转形态**:

| 形态 | 识别条件 | 前提条件 | 得分 |
|---|---|---|---|
| 锤子线 | 下影线 >= 2x 实体，上影线 <= 0.3x 实体 | 出现在下跌趋势中 | +1 ~ +2 |
| 倒锤子 | 上影线 >= 2x 实体，下影线 <= 0.3x 实体，实体为阳 | 出现在下跌趋势中 | +1 |
| 早晨之星 | (1)大阴线 (2)十字星或小实体 (3)大阳线，收盘深入(1)实体50%+ | 出现在下跌趋势末期 | +2 ~ +3 |
| 看涨吞没 | (1)阴线 (2)阳线，(2)实体完全包含(1)实体 | 出现在下跌趋势中 | +2 |
| 刺透形态 | (1)阴线 (2)阳线，(2)开盘低于(1)最低价，收盘超过(1)实体中点 | 出现在下跌趋势中 | +1 ~ +2 |
| 三白兵 | 连续3根阳线，每根开盘在前根实体内，收盘创新高 | 下跌趋势反转 | +2 ~ +3 |
| 跳空向上 | 当日最低价 > 前日最高价 | 下跌反转或突破确认 | +1 ~ +2 |

**看空反转形态**:

| 形态 | 识别条件 | 前提条件 | 得分 |
|---|---|---|---|
| 上吊线 | 形态同锤子线 | 出现在上涨趋势末期 | -1 ~ -2 |
| 射击之星 | 上影线 >= 2x 实体，下影线 <= 0.3x 实体，实体为阴 | 出现在上涨趋势末期 | -1 ~ -2 |
| 黄昏之星 | (1)大阳线 (2)十字星或小实体 (3)大阴线，收盘深入(1)实体50%+ | 出现在上涨趋势末期 | -2 ~ -3 |
| 看跌吞没 | (1)阳线 (2)阴线，(2)实体完全包含(1)实体 | 出现在上涨趋势中 | -2 |
| 乌云盖顶 | (1)阳线 (2)阴线，(2)开盘高于(1)最高价，收盘低于(1)实体中点 | 出现在上涨趋势中 | -1 ~ -2 |
| 三只乌鸦 | 连续3根阴线，每根开盘在前根实体内，收盘创新低 | 上涨趋势反转 | -2 ~ -3 |
| 跳空向下 | 当日最高价 < 前日最低价 | 上涨反转或破位 | -1 ~ -2 |

**中性/犹豫形态**:

| 形态 | 识别条件 | 含义 | 得分 |
|---|---|---|---|
| 十字星 | abs(close - open) <= 0.05 * (high - low) | 趋势末端=可能反转 | 0 (方向视位置) |
| 纺锤线 | 实体 < 全幅 * 0.2，上下影线均较长 | 多空犹豫 | 0 |
| 十字胎 | (1)大实体 (2)十字星，(2)完全在(1)实体范围内 | 趋势可能反转 | 0 |
| 孕线 | (1)大实体 (2)小实体，(2)完全在(1)实体范围内 | 趋势减弱 | 0 |

**K线形态得分计算规则**:
1. 从最新K线向前扫描，优先检测多根K线组合形态（早晨/黄昏之星 > 吞没 > 其他）
2. 单一K线形态（锤子/射击之星/十字星）作为辅助信号
3. 若多个形态信号方向一致，得分可叠加但不超过 +3 / -3
4. 若形态信号方向冲突，以组合形态优先，降低单根K线形态权重
5. 必须结合趋势位置判断（下跌末期的锤子线有效，上涨末期的锤子线无效）

详细识别标准见 [kline-patterns.md](kline-patterns.md)。

### 1.8 K线数据验证

**检查项**: 使用 Tushare 获取的 OHLCV 数据交叉验证技术指标

| 验证项 | 方法 | 作用 |
|---|---|---|
| 均线验证 | Tushare 返回的 ma5/ma10/ma20/ma60 与 close 计算值对比 | 确认均线信号准确性 |
| 量价确认 | 对比 vol 趋势与 close 趋势 | 验证放量上涨/缩量下跌判断 |
| 价格区间 | 根据 high/low 确定支撑/压力位 | 提供操作建议区间的数据依据 |
| 涨跌幅验证 | pct_chg 与 close/pre_close 计算值对比 | 确认数据一致性 |

K线数据验证不单独计分，但影响技术面各子项的置信度：
- 验证通过 → 维持原得分
- 验证不一致 → 降级得分绝对值 1 级（如 +2 降为 +1）
- 无K线数据 → 各子项得分不变但置信度标记为"低"
